## Task Planning Framework

Use this when the user asks for a plan, roadmap, implementation breakdown, migration path, or sequencing decision.

1. Restate the goal in one sentence.
2. Identify the smallest useful first milestone.
3. Break work into ordered steps with clear completion signals.
4. Call out risks, unknowns, and rollback options.
5. Include a verification step for each meaningful change.
6. If an AIRUI artifact would help, render a compact task table or timeline.

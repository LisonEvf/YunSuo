# 市场情绪策略看板 — 优化后能力评估报告

**评测角色：** 情绪短线炒家（涨停板战法、情绪周期、板块轮动为核心风格）
**评测日期：** 2026/06/03
**项目版本：** V3.0 PRO（优化后）
**对比基准：** 优化前首轮评测

---

## 一、总体评价：★★★★★ (4.5/5，较首轮 4/5 提升)

经过一轮系统性优化，项目从"复盘利器但盘中难用"升级为**盘前+盘中+盘后全场景可用的实战工具**。核心提升在三个维度：交互层从静态展示变为可交互、策略引擎从粗糙变精细、后端从单线程阻塞变为并行+缓存。最大的质变是 Agent 对话面板的接入——炒家盘中可以直接问"这个票能不能打"，这是从"看板"到"武器"的跨越。

| 类别 | 指标 | 状态 | 炒家实战价值 |
|------|------|------|------------|

| **晋级效率** | 连板晋级率（本轮新增） | ✅ **新增** | 昨日涨停股今日存活率，接力核心指标 |

**炒家评语：** 核心短线指标覆盖率约 90%，缺失的竞价数据是最大遗憾——这是盘中最重要的决策窗口。但考虑到数据源限制，已是在现有条件下的最优解。

## 三、策略分析引擎评估 ★★★★★（从 4 星升至 5 星）

### 3.1 情绪周期量化 ★★★★★

9 级细化标签 + 6 级粗标签映射，是目前同类自建工具中最精细的周期模型：

```
冰冰点 → 冰点 → 背离 / 耦合 → 退潮 → 常态 → 启动 → 发酵 → 高潮
```

**炒家价值：** "背离"和"耦合"两个细分标签对拐点判断极其重要——短线炒家最怕的不是退潮，而是**不知道退潮何时结束、启动是否确认**。这两个标签直接解决了这个问题。

### 3.2 手法评分 ★★★★☆（显著提升）

6 种手法（空仓/超跌反弹/低吸半路/首板打板/龙头接力/高位打板）各有量化得分。

**本轮改进：**
- ✅ 新增大盘趋势惩罚因子：`market_penalty = max(0, -avg_index_pct) * 12`，大盘跌 1% 时追高类手法被惩罚 12 分，**这是非常实战的改进**——很多炒家亏钱就是在指数大跌日强行追板
- ⚠️ 系数仍为经验公式，未经统计回测验证。建议长期积累实盘数据后做回归验证

**手法评分实战对照表：**

| 手法 | 核心变量 | 判断逻辑 | 实战准确度评估 |
|------|---------|---------|-------------|
| 空仓观望 | 炸板率+跌停+情绪反分+大盘惩罚 | ≥60 推荐空仓 | ★★★★★ 非常准确，"信号不足时空仓就是策略" |
| 超跌反弹 | 情绪反分-跌停+封板质量 | ≥50 可做 | ★★★★☆ 冰点修复初期确实适合轻仓试错 |
| 低吸半路 | 情绪+封板质量+溢价 | ≥55 可做 | ★★★★☆ 依赖主线确认，公式正确但需人工辅助判断 |
| 首板打板 | 涨停-炸板+封板质量-大盘惩罚 | ≥55 可做 | ★★★★☆ 涨停多+炸板少确实适合打板 |
| 龙头接力 | 情绪+溢价-跌停-大盘惩罚 | ≥55 可做 | ★★★☆☆ 最难量化的手法，高潮期容易失真 |
| 高位打板 | 情绪-炸板-跌停-大盘惩罚（最重） | ≥50 观察 | ★★★☆☆ 高位打板风险最大，公式惩罚力度合理 |

### 3.3 仓位建议 ★★★★★（从 4 星升至 5 星）

| 情绪区间 | 激进型 | 稳健型 | 评价 |
|----------|--------|--------|------|
| 跌停≥40 或 炸板率≥45 | 0-1成防守 | 空仓等待 | ✅ 极端行情保护 |
| <35（冰点/退潮） | 1成试错 | 0-1成 | ✅ 只做试错 |
| 35-60（常态） | 1-3成试错 | 1-2成 | ✅ 适度参与 |
| 60-80（启动/发酵） | 3-5成跟随 | 2-4成 | ✅ 主升浪仓位 |
| ≥80（高潮） | **1-2成仅核心** | **不追加速** | ✅ **本轮修正**，从 max:35 降至 max:20，更符合实战 |

**炒家评语：** 高潮期仓位收缩到 max:20 是关键改进。实际炒家在情绪≥85 的兑现日，应该只留核心仓位甚至空仓，之前的 max:35 偏激进。现在的建议可直接照做。

### 3.4 风险识别 ★★★★★（从 4 星升至 5 星）

**原有 4 维：** 跌停扩散 / 炸板率偏高 / 情绪高潮兑现 / 急速回撤

**本轮新增 3 维：**
- ✅ **跌停连续增加** — 近3日跌停递增且≥10，亏钱效应扩散趋势，**极其重要**
- ✅ **连板空间骤降** — 最高板≤2 且涨停减半，接力亏钱效应炸裂
- ✅ **主线退潮信号** — 主线板块转跌且涨停≤1，情绪拐头预警

**炒家评语：** 新增的三个风险维度都是实战中容易忽略但杀伤力极大的信号。特别是"跌停连续增加"——很多炒家只看当天的跌停数，不注意趋势，这个指标直接解决了这个盲点。

### 3.5 机会输出 ★★★★☆（从 3 星升至 4 星）

从只输出 1 条改为输出 TOP3 板块各 1 条，含差异化描述（板块名+强度+龙头+阶段+触发条件）。

**仍可改进：** 每条机会可以附带具体的观察标的和价位参考，而非仅板块级别的方向建议。

### 3.6 观察池 ★★★★☆（从 2 星升至 4 星）

**本轮改进：**
- 买点条件从固定文本"放量回封且板块共振"改为根据个股特征差异化生成
- 连板≥3 的标的：竞价不低开，放量换手后接力
- 连板 1-2 板的标的：板块共振且竞价正向
- 首板标的：首板放量回封确认
- 匹配到主线板块的额外加条件"XX板块维持强势"

**仍可改进：** 缺少具体价位参考（如"XX元上方回封"），仅文字条件仍偏抽象。

## 五、技术架构评估 ★★★★★（从 4 星升至 5 星）

### 5.1 本轮架构改进

| 改进 | 实现 | 效果 |
|------|------|------|
| **并行拉取** | `ThreadPoolExecutor(max_workers=4)`，11个数据源并行 | 首次加载耗时从串行累加降为最慢的单次 |
| **连接池复用** | `connections.py` KplClient/TdxClient 单例 | 每次 dashboard 省去 3-5 次 TCP 握手 |
| **SQLite 缓存** | `cache.py`，历史日仅拉取一次 | 30天趋势数据，第2次起只拉当天 1 天 |
| **轻量刷新接口** | `/api/dashboard/trend` | 前端自动刷新只拉趋势，不重新聚合全量 |
| **前端零框架** | 保持不变，纯 CSS 约 420 行 | 渲染极快，无框架开销 |

### 5.2 数据流架构（优化后）

```
前端 10秒轮询 ──→ /api/dashboard/trend（轻量，只取 trend[-N:]）
                        │
                        ▼
                   TTLCache（45秒）
                        │
                   cache miss
                        ▼
              SQLite 趋势缓存（历史日直接读库）
                        │
                   未缓存天 + 当天
                        ▼
              ThreadPoolExecutor 并行拉取
              ┌────────┬────────┐
              │ KPL ×9 │ TDX ×2 │  ← 单例连接池复用
              └────────┴────────┘
                        │
                   _normalize_dashboard
                        │
                   完整 DashboardData
```

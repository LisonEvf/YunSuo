## Code Review Framework

Use this when the user asks for review, audit, risk assessment, or validation of a change.

1. Prioritize concrete findings over summaries.
2. Reference files and lines when possible.
3. Focus on behavior, tests, safety, performance, and maintainability.
4. Separate confirmed issues from open questions.
5. If no issues are found, say so and name any remaining test gaps.
6. Render a review table only when there are multiple findings.

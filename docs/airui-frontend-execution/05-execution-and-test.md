# AIRUI 前端执行方案 05：执行与测试

## 执行顺序

1. 写入本方案文档。
2. 后端补 `/api/config` 读写接口和 Agent 重建能力。
3. 重写 `backend/static/airui/index.html`：
   - 固定左侧对话栏。
   - 完整右侧看板工作台。
   - Markdown 渲染。
   - 交互焦点面板。
   - 设置页。
4. 运行后端测试。
5. 重启本地服务。
6. 使用内置浏览器检查：
   - 页面可见。
   - 左右滚动独立。
   - 看板模块齐全。
   - 点击意图有效。
   - 对话 Markdown 和右侧 AI 研判同步。
   - 设置页可读写。

## 自动化测试

后端测试：

```bash
cd backend && python -m pytest tests -v
```

浏览器验证：

- 打开 `http://127.0.0.1:8000/dashboard/`
- 检查 DOM 是否包含：
  - `市场情绪工作台`
  - `情绪仪表盘`
  - `明日策略`
  - `AI 研判`
  - `设置`
  - `板块情绪热力分布`
  - `明日观察池`

## 风险与约束

- 当前正式 React/AIRUI 构建链仍受 AIRUI submodule 缺失影响，因此本阶段继续使用静态 HTML 作为可运行前端。
- MCP 本阶段提供配置存储，不接入 Agent 工具执行。
- 后端 drilldown 依赖 LLM 响应质量和模型可用性；前端点击先做即时反馈，避免等待 LLM。

## 验收标准

- 服务可通过 `/health`。
- 看板入口非空白。
- 左侧对话 100% 视口高度，右侧单独滚动。
- 主题、LLM、Skill、MCP 设置可见且可操作。
- 对话成功返回后，左侧 Markdown 可读，右侧 AI 研判同步更新。

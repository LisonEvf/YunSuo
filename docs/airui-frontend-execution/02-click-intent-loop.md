# AIRUI 前端执行方案 02：点击意图与循环交互

## 目标

看板组件不只是展示数据，还要表达“我可以被追问”。每个核心模块都提供基础点击意图，点击后形成：

看板元素 → 焦点面板 → 推荐追问 → 对话请求 → 右侧 AI 研判更新 → 新一轮点击。

## 交互对象

- KPI 卡片：解释指标、查看风险、进入策略。
- 趋势卡片：解释三线背离、查看近 15 日结构。
- 板块排行：查看板块详情、生成板块分析追问。
- 赚钱手法：查看适用条件、追问执行边界。
- 风险/机会：追问触发条件、加入明日预案。
- 明日观察池：查看个股买点条件、追问个股计划。
- 核心指数：查看指数环境对短线的影响。

## 前端行为

1. 所有可交互组件添加 `data-intent`、`data-kind`、`data-payload`。
2. 点击后更新右侧 `focusPanel`，展示：
   - 当前对象标题。
   - 对象关键字段。
   - 可执行动作按钮。
   - 推荐追问 prompt。
3. 如果对象来自 AIRUI 表格，前端同时通过 WebSocket 发送 `interaction: "drilldown"`，保留后端现有 drilldown 能力。
4. 推荐追问按钮直接填充并发送到左侧对话，形成循环。

## 后端行为

后端已有 `/ws/airui` interaction 入口和 `drilldown.py`。本阶段保留该能力：

- 表格行点击发送 `drilldown`。
- 后端可调用 Agent 和 `render_airui_panel` 更新看板。
- 前端收到新 document 后重新渲染，并保留当前焦点。

## 交付标准

- 点击 KPI、板块、风险、观察池任一项，右侧能出现焦点解释。
- 焦点解释中至少提供 2 个可执行追问按钮。
- 表格类点击会发送 WebSocket drilldown，不阻塞前端反馈。

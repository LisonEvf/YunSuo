# AIRUI 前端执行方案 03：对话 Markdown 与看板同步

## 目标

对话返回时，左侧支持 Markdown 阅读，右侧同步形成“AI 研判面板”。右侧作为主表达，左侧作为上下文、补充解释与追问入口。

## 对话渲染

左侧消息渲染需要支持：

- 标题 `#` / `##` / `###`
- 加粗、斜体、行内代码
- 无序列表、有序列表
- 引用块
- Markdown 表格
- 换行段落

由于当前静态页面不依赖构建链，本阶段使用轻量 Markdown 渲染器，先做安全转义，再转换常用语法。

## 看板同步

对话请求完成后，前端执行：

1. 左侧追加用户消息。
2. 左侧显示 Agent 思考状态。
3. 请求 `/api/chat`，携带设置中启用的 skills。
4. 得到返回后：
   - 左侧渲染 Markdown。
   - 右侧 `assistantPanel` 生成 AI 研判卡片。
   - 自动切换或提示进入“AI 研判”视图。
5. 如果 SSE 或工具事件返回 AIRUI 内容，则优先将 AIRUI 结果作为右侧主表达；普通文本作为辅助解释。

## 主表达结构

AI 研判面板包含：

- 原始问题。
- 结论摘要。
- Markdown 渲染内容。
- 关联看板指标快照。
- 下一步追问按钮。

## 交付标准

- 左侧 AI 回复不再是纯文本，Markdown 表格可读。
- 右侧出现与本次问题绑定的 AI 研判区域。
- 追问按钮能继续触发 `/api/chat`，并更新右侧主表达。

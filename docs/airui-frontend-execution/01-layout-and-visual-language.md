# AIRUI 前端执行方案 01：布局与视觉语言

## 目标

将当前临时看板从“能看到数据”升级为“以看板为交互起点、以对话为辅助解释”的工作台。整体布局采用左右双栏：

- 左侧为固定高度对话栏，占据视口 100% 高度，内部消息独立滚动，不随右侧看板滚动。
- 右侧为主看盘区域，独立滚动，默认展示情绪仪表盘。
- 设置、主题、刷新、视图切换放在右侧顶部工具栏，避免把主表达挤到左侧对话里。

## 设计指导思想

参考 Apple UI 的核心不是表面拟态，而是：

- 清晰层级：用标题、局部摘要、主数据、辅助解释建立阅读路径。
- 直接操控：卡片、表格行、趋势、观察池都应显示可点击状态，并在点击后给出明确反馈。
- 材料感克制：使用浅色默认主题、柔和边界、细阴影和半透明状态层；深色主题保留但不成为唯一视觉。
- 让界面也参与对话：右侧看板是“主回答”，左侧对话提供补充语义和追问入口。

## 布局方案

1. `body` 固定为 `height: 100vh; overflow: hidden`。
2. `.app-shell` 使用 `grid-template-columns: 380px minmax(0, 1fr)`。
3. `.conversation-rail` 使用 `height: 100vh; display: flex; flex-direction: column`。
4. `.workspace` 使用 `height: 100vh; overflow: auto`，所有看板内容只在右侧滚动。
5. 小屏幕下改为上下布局，对话栏最大高度约 42vh，看板仍为主要起点。

## 看板信息架构

右侧保留原 Vue 前端已有模块，并按“先结论、再结构、再执行”的顺序组织：

- 顶部：交易日、周期、连接状态、刷新状态。
- 情绪仪表盘：情绪综合指数、最新交易日、涨停/炸板/跌停、炸板率。
- 状态条：大盘系数、超短情绪、赚钱效应、三线分歧、大盘 VS 超短、交易建议。
- 主视图：三线趋势、风险摘要、板块强度、板块热力。
- 策略视图：周期定位、核心指数、板块梯队、赚钱手法、风险 Checklist、机会 Watchlist、明日观察池。
- AI 研判视图：对话返回后的主表达区域。
- 设置视图：主题、LLM、Skill、MCP 配置。

## 交付标准

- 左侧对话栏不随右侧滚动。
- 右侧默认进入“情绪仪表盘”，而不是欢迎页或空白页。
- 原 Vue 前端的重要功能模块均能在当前页面找到对应入口。
- 主题切换后全局颜色、卡片、输入框、表格、按钮同步变化。
